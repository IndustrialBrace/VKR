## Извлечённые тезисы для ВКР

### Главный тезис для 2.2 — ПРЯМОЙ конкурент Face2Pass

В отличие от VisionLabs (источник № 37), который занимает технологический слой и продаёт алгоритмы и платформы, **OVISION — прямой продуктовый конкурент Face2Pass** в сегменте биометрических СКУД для корпоративных и государственных клиентов. Они решают одну и ту же задачу — контроль доступа сотрудников по лицу — и формально подходят к одной и той же когорте заказчиков.

### Природа конкуренции в сегменте СКУД

Прямое столкновение OVISION ↔ Face2 возникает в **B2B/B2G-сегменте контроля доступа сотрудников по лицу**. Здесь у двух игроков диаметрально разные сильные стороны:

| Преимущество | OVISION | Face2 |
|---|---|---|
| Производство собственных терминалов | Да | Нет |
| Полный набор формальных аккредитаций (Минцифры + Минпромторг + реестр ПО + ЕБС + СКЗИ) | Да | Частично |
| Единый поставщик «под ключ» | Да | Да (КБС «под ключ») |
| Доказанные референсы федеральных госкомпаний (РЖД и т. п.) | Да | Нет (на момент 2024–2025) |
| Банковская экосистема (платёжные сервисы, мобильное приложение клиента, точки продаж в Татарстане) | Нет | Да |
| Финансово-сервисная вертикаль (Pay → Pass → Check-in внутри одного банка) | Нет | Да |
| Право регистрировать БПд граждан в ЕБС (как банк) | Нет (только корпоративная аутентификация) | Да (ПАО «АК БАРС» БАНК — № 101 в перечне ФЗ-115) |

### Стратегические следствия для главы 3

1. **OVISION закрывает «классический рынок СКУД»** — продаёт то, что банк продать не может (терминал собственного производства + формальные статусы Минцифры/Минпромторга/СКЗИ). Прямая конкуренция за крупных корпоративных клиентов с традиционным запросом на СКУД для Face2 невыгодна без собственного оборудования.
2. **Face2 имеет устойчивую дифференциацию** — банковская инфраструктура и право работать с физическими лицами через ЕБС. Единственный конкурент, у которого этого нет, и есть OVISION (так как банком не является).
3. **Возможный сценарий — кооперация, а не конкуренция.** Face2 может использовать терминалы OVISION (как СКУД-уровень железа) и связывать их с банковской аутентификацией через ЕБС. Это снимет необходимость собственного производства и закроет требование Минпромторга/Минцифры по статусу.
4. **Целевые сегменты для Face2 в B2B/B2G:** не «общие СКУД для госструктур» (там доминируют OVISION и подобные производители), а **банковско-связанные сценарии** — корпоративный банкинг, расчётные центры, бизнес-залы аэропортов, гостиничные сети с финансовой интеграцией.
